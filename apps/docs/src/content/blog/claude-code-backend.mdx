Claude Code 시리즈 6편이에요. [5편에서 MCP로 노션·피그마를 연동](/blog/claude-code-mcp)했는데요.

오늘은 좀 부끄러운 이야기예요.

저 사실 **백엔드 잘 몰라요.** 프론트엔드 개발자거든요. Spring이라고 하면 어노테이션 몇 개 아는 수준이고, MyBatis는 이름만 들어봤고, JWT 구현은 블로그 긁어서 붙여넣는 사람이에요.

근데 지금 hanui CMS 백엔드를 짜고 있어요. Spring Boot + 전자정부프레임워크 v5로.

어떻게 된 일이냐면...

## 전자정부프레임워크가 뭔지도 몰랐어요

공공기관 프로젝트 들어가면 꼭 나오는 그 이름. 행정안전부에서 만든 Java 기반 프레임워크예요.

쉽게 말하면 **공공기관 납품용 블록 모음**이에요.

```
eGovFramework = Spring Boot 기반
               + 인증/세션 처리 블록
               + 파일 업로드 블록
               + 공통 코드 관리 블록
               + 게시판 블록
               + 로그 블록
               + ...
```

HANUI가 UI 컴포넌트 블록 모음인 것처럼, 전자정부프레임워크는 **백엔드 기능 블록 모음**이에요.

공공기관 입찰할 때 "전자정부프레임워크 적용"이 거의 필수 요건이라 쓸 수밖에 없기도 하고요.

저는 이게 뭔지도 모르는 상태에서 Claude Code한테 물어봤어요.

```
나: 전자정부프레임워크가 뭐야?
    공공기관 CMS 만들어야 하는데 이걸 써야 한다고 하더라고

Claude: (설명해줌)
        → 어떤 블록이 있는지
        → Spring Boot 3.x와 어떻게 쓰는지
        → 프로젝트에 어떻게 세팅하는지
```

이게 시작이었어요.

## "몰라도 된다"는 걸 알았어요

솔직히 말하면, 저는 코드를 이해하면서 짜는 게 아니에요.

Claude Code가 코드를 만들어주고, 저는 이게 뭘 하는 건지 물어가면서 이해하는 식이에요.

```
나: 이 @PreAuthorize("hasRole('EDITOR')") 이게 뭐야?

Claude: Spring Security에서 이 API를
        EDITOR 역할을 가진 사람만 접근할 수 있게 막는 거예요.
        RBAC(역할 기반 접근 제어)라고 해요.

나: 아 그럼 VIEWER는 이 API 못 쓰겠네?

Claude: 맞아요. VIEWER는 읽기만 되고 쓰기는 안 돼요.
```

이게 반복되다 보니까 어느새 대충 흐름은 알게 됐어요.

## 실제로 이렇게 했어요

### 1. 노션 기획서 → DB 스키마

노션에 DB 설계가 다 되어있었어요. 테이블, 컬럼, 관계 전부요.

```
나: 노션 기획서 DB 설계 페이지 보고
    PostgreSQL DDL 만들어줘
    Flyway 마이그레이션 파일로

Claude: (노션 직접 읽음)
        → V1__init_schema.sql 생성
```

저는 SQL을 잘 몰라도 기획서만 써놓으면 됐어요.

### 2. API 명세 → 코드

API 명세도 노션에 있었어요.

```
나: 게시글 CRUD API 만들어줘
    명세는 노션에 있어

Claude: PostController.java
        PostService.java
        PostMapper.java + PostMapper.xml
        PostDto.java (Request/Response)
        → 전부 생성
```

저는 "이 메서드가 뭘 하는 거야?"를 계속 물어가면서 배웠어요.

### 3. 모르는 거 나오면 바로 질문

```
나: @Transactional이 여기 왜 붙어있어?

Claude: 게시글 저장하고 첨부파일도 같이 저장하는데,
        둘 중 하나 실패하면 둘 다 취소해야 하거든요.
        그게 트랜잭션이에요.

나: 아 그래서 같이 묶는 거구나
```

이런 식으로 하다 보니까 Spring을 하나씩 배우고 있어요.

## 블록 조합하는 거잖아요, 결국

프론트엔드에서 컴포넌트 조합하는 것처럼, 백엔드도 결국 블록 조합이더라고요.

```
전자정부프레임워크 블록
  + Spring Security (인증/권한)
  + MyBatis (DB 쿼리)
  + Flyway (DB 마이그레이션)
  + Claude Code (조합 + 구현)
```

저는 어떤 블록을 어떻게 조합할지 **설계**만 했어요. 구현은 Claude Code가 했고, 모르는 건 Claude Code한테 물어봤어요.

## 근데 양심이 좀 찔리긴 해요

글 쓰면서 느끼는 게, 제가 뭔가 대단한 걸 한 것처럼 되는데... 사실 저는 설계하고 지시한 거고 구현은 Claude Code가 다 했어요.

그냥 솔직하게 말하면:

> 백엔드 잘 모르는 프론트엔드 개발자가
> 노션에 기획서 잘 써두고
> Claude Code한테 시켰어요.

근데 그게 어때서요? 어차피 돌아가면 되는 거 아닌가요.

"이해하지 못한 코드는 쓰면 안 된다"고 하는 사람도 있을 것 같은데, 저는 모르는 코드가 나오면 물어보면서 이해했어요. 그게 Stack Overflow 긁어다 붙여넣는 것보다 훨씬 이해가 잘 됐어요.

## 결국 이게 요점인 것 같아요

Claude Code 이전에는:
- "나는 백엔드 모르니까 못 해"
- "백엔드 개발자 구해야겠다"

Claude Code 이후에는:
- "일단 해봐, 모르면 물어보면서"
- "기획이 명확하면 구현은 따라온다"

할 수 있는 것의 범위가 넓어졌어요.
