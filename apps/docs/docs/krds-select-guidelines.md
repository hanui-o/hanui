# KRDS Select(선택 목록) 가이드라인

> 출처: KRDS (한국형 웹 콘텐츠 접근성 지침)
>
> HANUI Select 컴포넌트는 이 가이드라인을 준수합니다.

## 선택 목록이란?

선택 목록(Select/Dropdown)은 여러 옵션 중 하나 또는 여러 개를 선택할 수 있도록 제공되는 인터랙션 요소입니다. 공간을 효율적으로 사용하면서 많은 선택지를 제공할 수 있는 컴포넌트입니다.

## 구조

1. **레이블(Label)** - 선택 목록의 목적을 설명하는 텍스트
2. **선택 버튼(Trigger)** - 옵션 목록을 여는 버튼
3. **플레이스홀더(선택)** - 선택 전 표시되는 힌트 텍스트
4. **옵션 목록(Dropdown Menu)** - 선택 가능한 항목들의 리스트
5. **선택된 값 표시** - 현재 선택된 값을 보여주는 영역
6. **화살표 아이콘(선택)** - 드롭다운을 나타내는 시각적 표시

---

## 용례

### ✅ 사용하기 적합한 경우

옵션이 5개 이상이고, 사용자가 하나 또는 여러 개를 선택해야 할 때

- 국가, 도시 등 많은 선택지가 있는 경우
- 카테고리, 분류 등을 선택할 때
- 정렬 옵션을 선택할 때 (최신순, 인기순 등)
- 기간, 날짜 범위를 선택할 때

### ❌ 사용하기 적합하지 않은 경우

선택지가 적거나 항상 보여야 하는 경우

- 선택지가 3개 이하인 경우 → Radio Button 사용
- 사용자가 모든 옵션을 한눈에 비교해야 하는 경우 → Radio Button 사용
- 텍스트 입력이 필요한 경우 → Input with Autocomplete 사용
- 여러 개를 선택하고 선택 상태를 명확히 봐야 하는 경우 → Checkbox 사용

## 사용성 가이드라인

### 1. 레이블은 명확하게 제공한다

모든 선택 목록에는 무엇을 선택하는지 명확히 알려주는 레이블이 필요합니다. "선택하세요"가 아닌 "배송 국가를 선택하세요"처럼 구체적으로 작성합니다.

### 2. 옵션은 논리적 순서로 정렬한다

- 알파벳순, 가나다순 등 예측 가능한 순서
- 사용 빈도가 높은 순서
- 중요도 순서

단, 무작위 순서는 사용자 혼란을 야기하므로 피합니다.

### 3. 기본값은 신중하게 선택한다

기본값이 있다면 가장 일반적이거나 안전한 선택지로 설정합니다. 불확실한 경우 플레이스홀더만 표시하고 사용자가 명시적으로 선택하도록 합니다.

### 4. 옵션 개수가 많을 경우 검색 기능을 제공한다

옵션이 20개 이상인 경우, 드롭다운 내 검색 기능을 추가하여 사용자가 빠르게 원하는 항목을 찾을 수 있도록 합니다.

### 5. 모바일 환경을 고려한다

모바일에서는 네이티브 선택 인터페이스(하단 시트, 모달 등)를 사용하는 것이 더 나은 사용자 경험을 제공할 수 있습니다.

### 6. 비활성화(Disabled) 옵션 사용을 최소화한다

비활성화된 옵션은 혼란을 줄 수 있으므로, 가능하면 아예 표시하지 않거나 왜 선택할 수 없는지 설명을 제공합니다.

### 7. 다중 선택 시 선택된 항목을 명확히 표시한다

다중 선택 모드에서는 체크박스를 함께 표시하고, 선택된 항목 개수나 태그 형태로 현재 선택 상태를 보여줍니다.

## 접근성 체크리스트

- [ ] 모든 선택 목록에 label이 연결되어 있는가?
- [ ] 키보드로 열기/닫기/탐색이 가능한가? (Enter/Space, Arrow keys, Esc)
- [ ] 포커스 상태가 명확하게 표시되는가?
- [ ] aria-haspopup="listbox" 속성이 적용되어 있는가?
- [ ] 열린 상태에서 aria-expanded="true"가 설정되는가?
- [ ] 선택된 항목에 aria-selected="true"가 있는가?
- [ ] 필수 선택 항목에 required 또는 aria-required가 있는가?
- [ ] 에러 상태 시 aria-invalid와 aria-describedby가 연결되는가?
- [ ] 색상 대비가 WCAG AA 기준(4.5:1)을 충족하는가?
