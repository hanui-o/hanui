# Button 컴포넌트 문서 페이지 개선안

> KRDS 가이드라인을 통합한 공공 웹 전용 Button 문서 구조

## 목표

- **차별화**: shadcn/ui와 달리 KRDS 준수 가이드라인 제공
- **교육적**: 왜 이렇게 써야 하는지, 언제 써야 하는지 명확히 제시
- **실용적**: 실제 공공기관 웹사이트 개발에 즉시 적용 가능

### 3. **🆕 개요 (Overview)**

```markdown
## 개요

버튼은 사용자가 서비스를 이용하는 과정에서 어떤 행동이 중요한지 알려주는
핵심 인터랙션 요소입니다. HANUI Button은 KRDS를 준수하여 공공 웹사이트에
최적화된 접근성과 사용성을 제공합니다.
```

**포함 내용:**

- 버튼의 역할과 중요성 (1-2문장)
- KRDS 준수 명시
- 접근성 강조

### 4. **🆕 사용 가이드라인 (Usage Guidelines)**

이 섹션이 **핵심 차별화 포인트**입니다!

#### 4-1. 언제 사용해야 하나요?

```markdown
### 언제 사용해야 하나요?

#### ✅ 버튼을 사용하기 적합한 경우

- **폼 제출이나 대화창 실행** - 입력폼 제출, 대화창 실행, 기능 취소 등
- **상태 전환** - 도움말 제공, 중요한 데이터 완전히 삭제
- **중요한 액션** - 전행 중인 프로세스 중단/취소

[실제 예시 이미지/코드]

#### ❌ 버튼을 사용하지 말아야 하는 경우

- **페이지 이동** - 다른 화면이나 서비스로 이동 시 → Link 컴포넌트 사용

[잘못된 예시 vs 올바른 예시]
```

**구현 방식:**

- 녹색 박스(✅): 올바른 사용 사례
- 빨간색 박스(❌): 잘못된 사용 사례
- 각 케이스마다 실제 코드 예제 포함

#### 4-2. 버튼 위계 가이드

```markdown
### 버튼 위계 (Button Hierarchy)

버튼의 시각적 강조도는 액션의 중요도와 일치해야 합니다.

#### Primary - 가장 중요한 액션

- 페이지당 하나만 사용 권장
- 주요 목표 달성 액션 (제출, 저장, 구매 등)

[Primary Button 예시]

#### Secondary - 보조 액션

- Primary와 함께 사용
- 취소, 이전 단계 등

[Secondary Button 예시]

#### Outline - 낮은 강조

- 추가 옵션이나 덜 중요한 액션

[Outline Button 예시]

#### Ghost - 최소 강조

- 인라인 액션, 닫기 버튼 등

[Ghost Button 예시]
```

**시각적 표현:**

```
┌─────────────────────────────────┐
│ 강조도: High → Low              │
│                                 │
│ [Primary] > [Secondary] >       │
│ [Outline] > [Ghost]             │
└─────────────────────────────────┘
```

#### 4-3. 버튼 배치 가이드

```markdown
### 버튼 배치 (Button Placement)

#### ✅ 올바른 배치

**수평 배치 (기본)**

- 가장 중요한 버튼을 오른쪽에 배치
- [취소] [확인] ← 이 순서

**수직 배치**

- 가장 중요한 버튼을 위에 배치
- 한 페이지에 최상위 강조 버튼은 하나만
```

**예시 컴포넌트:**

```tsx
// 모범 사례
<div className="flex items-center gap-3">
  <Button variant="outline">취소</Button>
  <Button variant="primary">확인</Button>
</div>

// 피해야 할 사례
<div className="flex items-center gap-3">
  <Button variant="primary">취소</Button>
  <Button variant="primary">확인</Button> {/* ❌ Primary 2개 */}
</div>
```

#### 4-4. 레이블 작성 가이드

```markdown
### 버튼 레이블 (Button Labels)

#### 원칙

- **동사형으로 작성**: "저장", "제출", "삭제" (O) / "저장됨", "제출 완료" (X)
- **구체적으로 작성**: "확인" (X) → "회원가입 완료" (O)
- **간결하게 작성**: 2-3단어 이내 권장

#### ✅ 좋은 예시

- "파일 업로드"
- "댓글 작성"
- "다음 단계"

#### ❌ 나쁜 예시

- "OK"
- "클릭"
- "버튼"
```

#### 4-6. 접근성 고려사항

```markdown
### 접근성 (Accessibility)

#### 중복 클릭 방지

- 네트워크 지연 시 사용자가 버튼을 여러 번 클릭할 수 있음
- `loading` 상태 활용 권장

<Button loading disabled>처리 중...</Button>

#### 키보드 접근성

- 모든 버튼은 Tab 키로 포커스 가능
- Enter/Space로 실행 가능

#### 명도 대비

- WCAG AA 기준 충족 (4.5:1 이상)
- 모든 variant는 명도 대비 테스트 완료
```
