# KRDS Input(입력 필드) 가이드라인

> 출처: KRDS (한국형 웹 콘텐츠 접근성 지침)
>
> HANUI Input 컴포넌트는 이 가이드라인을 준수합니다.

## 입력 필드란?

입력 필드(Input Field)는 사용자가 텍스트나 숫자 등의 정보를 직접 입력할 수 있도록 제공되는 인터랙션 요소입니다. 폼(Form)의 핵심 구성 요소로, 사용자와 시스템 간의 정보 교환을 가능하게 합니다.

## 구조

1. **레이블(Label)** - 입력 필드의 목적을 설명하는 텍스트, 모든 입력 필드는 명확한 레이블을 제공해야 함
2. **입력 영역(Input Area)** - 사용자가 실제로 데이터를 입력하는 영역
3. **플레이스홀더(선택)** - 입력 예시를 보여주는 힌트 텍스트
4. **도움말 텍스트(선택)** - 입력 형식이나 제약사항을 설명하는 보조 텍스트
5. **에러 메시지(선택)** - 입력 값이 유효하지 않을 때 표시되는 피드백
6. **아이콘(선택)** - 입력 필드의 용도를 시각적으로 보조하는 요소

---

## 용례

### 사용하기 적합한 경우

사용자로부터 자유 형식의 텍스트나 특정 형식의 데이터를 입력받아야 할 때

- 이름, 주소, 이메일 등 텍스트 정보를 수집할 때
- 검색어를 입력받을 때
- 비밀번호, 전화번호 등 특수한 형식의 데이터를 받을 때
- 숫자나 날짜 등 구조화된 데이터를 입력받을 때

### 사용하기 적합하지 않은 경우

미리 정의된 선택지 중 하나를 선택해야 하는 경우

- 선택지가 5개 이하로 제한적인 경우 → Radio Button 사용
- 여러 옵션 중 하나 또는 여러 개를 선택하는 경우 → Select 또는 Checkbox 사용
- Yes/No와 같은 이진 선택인 경우 → Checkbox 또는 Toggle 사용

## 사용성 가이드라인

### 1. 레이블은 항상 제공한다

모든 입력 필드에는 명확한 레이블이 필요합니다. 레이블은 입력 필드 위 또는 왼쪽에 배치하며, 시각적으로 숨겨진 경우에도 스크린 리더를 위한 레이블은 반드시 제공해야 합니다.

### 2. 플레이스홀더는 레이블을 대체할 수 없다

플레이스홀더는 입력 예시를 보여주는 보조 수단일 뿐, 레이블을 대체할 수 없습니다. 사용자가 입력을 시작하면 플레이스홀더가 사라지므로 레이블은 별도로 제공되어야 합니다.

### 3. 적절한 input type을 사용한다

HTML5의 다양한 input type(email, tel, url, number, date 등)을 적절히 사용하여 모바일 환경에서 최적화된 키보드를 제공하고, 브라우저의 자동 검증 기능을 활용합니다.

### 4. 에러 메시지는 명확하고 구체적으로 제공한다

"잘못된 입력입니다"가 아닌 "이메일 형식이 올바르지 않습니다. '@'를 포함해주세요"와 같이 구체적인 해결 방법을 제시합니다.

### 5. 필수 입력 항목을 명확히 표시한다

필수 입력 항목에는 시각적 표시(\*, required 등)와 함께 aria-required 속성을 사용하여 스크린 리더 사용자에게도 전달합니다.

### 6. 입력 필드의 크기는 예상 입력 길이와 일치시킨다

우편번호처럼 짧은 입력은 짧은 너비로, 주소처럼 긴 입력은 충분한 너비로 제공하여 사용자가 입력할 내용의 길이를 예측할 수 있도록 합니다.

### 7. 자동완성(Autocomplete)을 적절히 활용한다

이름, 이메일, 주소 등 반복적으로 입력되는 정보에 대해 autocomplete 속성을 사용하여 사용자 편의성을 높입니다.

### 8. 포커스 상태를 명확히 표시한다

키보드로 탐색하는 사용자를 위해 포커스된 입력 필드는 명확한 시각적 표시(outline, border 변화 등)를 제공해야 합니다.

## 접근성 체크리스트

- [ ] 모든 입력 필드에 label이 연결되어 있는가?
- [ ] 에러 상태 시 aria-invalid="true"가 설정되는가?
- [ ] 에러 메시지가 aria-describedby로 연결되어 있는가?
- [ ] 필수 입력 항목에 required 또는 aria-required 속성이 있는가?
- [ ] 적절한 input type이 사용되고 있는가?
- [ ] 포커스 표시가 명확하게 보이는가?
- [ ] 색상 대비가 WCAG AA 기준(4.5:1)을 충족하는가?
- [ ] 최소 터치 영역(32px 이상)을 확보했는가?
