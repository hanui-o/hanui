# KRDS (한국형 웹 콘텐츠 디자인 시스템)

> 대한민국 정부 웹사이트의 접근성 표준 디자인 시스템

---

## 📖 개요

KRDS는 **"모든 사용자가 동등하게 이용할 수 있는 디지털 정부 서비스"**를 구현하기 위한 공식 디자인 시스템입니다.

### 준수 기준

KRDS의 모든 구성 요소는 다음 3가지 표준을 따릅니다:

1. **KWCAG 2.2** (한국형 웹 콘텐츠 접근성 지침 2.2)
2. **전자정부 웹사이트 품질관리 지침** (웹 접근성)
3. **WCAG 2.1** (국제 웹 접근성 표준)

---

## 🎯 핵심 원칙

### 디지털 포용 (Digital Inclusion)

모든 사용자를 포용하는 서비스 설계:

- 장애인 (시각, 청각, 신체, 인지)
- 고령자
- 어린이
- 외국인
- 저시력 사용자
- 색각이상자

### 4대 접근성 원칙

| 원칙              | 설명                                | 핵심 요구사항              |
| ----------------- | ----------------------------------- | -------------------------- |
| **인식의 용이성** | 정보를 이해할 수 있도록 제공        | 대체 텍스트, 명도 대비     |
| **운용의 용이성** | 키보드 및 보조기기로 조작 가능      | 키보드 접근성, 충분한 시간 |
| **이해의 용이성** | 콘텐츠를 쉽게 이해하고 사용 가능    | 명확한 지시사항, 오류 정정 |
| **견고성**        | 다양한 브라우저 및 보조 기술과 호환 | 웹 표준 준수, 마크업 오류  |

## ⚠️ 중요 제한사항

> **KRDS 적용만으로는 웹 접근성을 완벽히 충족할 수 없습니다.**

### 필수 추가 조치

1. **구축 과정에서의 지속적 모니터링**
   - 실제 스크린 리더 테스트
   - 키보드 전용 탐색 테스트
   - 명도 대비 검증

2. **운영 과정에서의 지속적 개선**
   - 사용자 피드백 수집
   - 최신 지침 업데이트 반영
   - 정기적인 접근성 감사

3. **콘텐츠 제작 시 주의사항**
   - 의미 있는 이미지 대체 텍스트
   - 명확한 링크 텍스트
   - 구조화된 제목 체계

## 🔧 HANUI 자동화 항목

HANUI는 다음 KRDS 요구사항을 자동으로 처리합니다:

### 자동 처리 항목 ✅

- **키보드 접근성**: `tabIndex`, `onKeyDown` 자동 추가
- **ARIA 속성**: `role`, `aria-label`, `aria-expanded` 자동 설정
- **초점 관리**: Focus Trap, Focus Restore
- **색상 대비**: Design Token에서 자동 검증
- **의미론적 마크업**: 적절한 HTML 태그 사용

### 수동 확인 필요 항목 ⚠️

- 이미지 `alt` 텍스트 작성 (의미 파악 필요)
- 링크 텍스트 명확성
- 콘텐츠 제목 구조화
- 언어 속성 (`lang`)
- 페이지 제목 (`<title>`)
