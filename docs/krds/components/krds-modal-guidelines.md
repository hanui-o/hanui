# KRDS Modal(모달) 가이드라인

> 출처: KRDS (한국형 웹 콘텐츠 접근성 지침)
>
> HANUI Modal 컴포넌트는 이 가이드라인을 준수합니다.

## 모달이란?

모달(Modal Dialog)은 사용자의 주의를 집중시키고 즉각적인 응답이나 결정을 요구할 때 사용하는 오버레이 형태의 인터랙션 요소입니다. 현재 작업 흐름을 중단하고 새로운 컨텍스트를 제공합니다.

## 구조

1. **오버레이(Overlay/Backdrop)** - 배경을 어둡게 만들어 모달에 집중하도록 함
2. **모달 컨테이너** - 실제 콘텐츠를 담는 영역
3. **제목(Title)** - 모달의 목적을 설명하는 헤더
4. **본문(Content)** - 주요 내용이나 폼
5. **액션 버튼(Actions)** - 확인, 취소 등의 버튼
6. **닫기 버튼(Close Button)** - 모달을 닫는 X 버튼

---

## 용례

### 사용하기 적합한 경우

사용자의 즉각적인 응답이 필요하거나 중요한 정보를 전달할 때

- 중요한 경고나 확인 메시지 (삭제, 취소 등)
- 추가 정보 입력을 위한 짧은 폼
- 이미지나 비디오 크게 보기
- 중요한 공지사항이나 약관 동의
- 복잡한 설정이나 필터 옵션

### 사용하기 적합하지 않은 경우

작업 흐름을 방해하거나 자주 나타나는 경우

- 광고나 마케팅 메시지
- 정보성 메시지 → Toast/Snackbar 사용
- 작은 도움말 → Tooltip 사용
- 복잡하고 긴 폼 → 별도 페이지로 이동
- 연속적인 모달 (모달 안의 모달)

## 사용성 가이드라인

### 1. 제목은 명확하고 구체적으로 작성한다

"알림"이 아닌 "게시글을 삭제하시겠습니까?"처럼 사용자가 무엇을 해야 하는지 명확히 전달합니다.

### 2. 액션 버튼은 명확한 레이블을 사용한다

"확인/취소" 대신 "삭제/보관" 처럼 실제 액션을 나타내는 동사를 사용합니다.

### 3. 백드롭 클릭으로 닫을 수 있어야 한다

단, 중요한 결정이 필요한 경우 (삭제 확인 등)는 명시적인 버튼 클릭만 허용할 수 있습니다.

### 4. ESC 키로 닫을 수 있어야 한다

키보드 사용자를 위해 ESC 키로 모달을 닫을 수 있어야 합니다.

### 5. 포커스 트랩을 구현한다

모달이 열리면 포커스가 모달 내부에 갇혀야 하며, Tab 키로 모달 밖으로 이동할 수 없어야 합니다.

### 6. 모달이 닫히면 이전 포커스로 돌아간다

모달을 연 버튼으로 포커스가 돌아가야 합니다.

### 7. 과도한 사용을 피한다

모달은 작업 흐름을 중단시키므로 꼭 필요한 경우에만 사용합니다.

### 8. 모바일 환경을 고려한다

모바일에서는 하단 시트(Bottom Sheet)가 더 나은 경험을 제공할 수 있습니다.

## 접근성 체크리스트

- [ ] role="dialog" 또는 role="alertdialog" 속성이 있는가?
- [ ] aria-labelledby로 제목과 연결되어 있는가?
- [ ] aria-describedby로 설명과 연결되어 있는가?
- [ ] aria-modal="true"가 설정되어 있는가?
- [ ] 포커스 트랩이 구현되어 있는가?
- [ ] ESC 키로 닫을 수 있는가?
- [ ] 모달이 열릴 때 첫 번째 포커스 가능한 요소로 포커스가 이동하는가?
- [ ] 모달이 닫힐 때 이전 포커스로 돌아가는가?
- [ ] 백그라운드 스크롤이 비활성화되는가?
- [ ] 색상 대비가 WCAG AA 기준(4.5:1)을 충족하는가?
