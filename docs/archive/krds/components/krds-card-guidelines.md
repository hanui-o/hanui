# KRDS Card(카드) 가이드라인

> 출처: KRDS (한국형 웹 콘텐츠 접근성 지침)
>
> HANUI Card 컴포넌트는 이 가이드라인을 준수합니다.

## 카드란?

카드(Card)는 관련된 정보를 그룹화하여 보여주는 컨테이너 컴포넌트입니다. 시각적으로 구분된 영역에 콘텐츠를 담아 스캔하기 쉽고 상호작용하기 쉬운 인터페이스를 제공합니다.

## 구조

1. **컨테이너** - 카드의 경계를 정의하는 영역
2. **이미지/미디어(선택)** - 시각적 콘텐츠
3. **제목** - 카드의 주요 내용을 요약
4. **본문** - 상세 설명이나 정보
5. **액션(선택)** - 버튼이나 링크

---

## 용례

### 사용하기 적합한 경우

여러 항목을 그리드나 리스트 형태로 보여줄 때

- 제품 목록
- 블로그 포스트 미리보기
- 사용자 프로필
- 대시보드 위젯
- 기능 소개

### 사용하기 적합하지 않은 경우

- 단일 정보만 표시하는 경우
- 복잡한 폼이나 긴 텍스트
- 너무 많은 정보를 한 카드에 담는 경우

---

## 사용성 가이드라인

### 1. 일관된 구조 유지

같은 컨텍스트의 카드들은 동일한 구조와 레이아웃을 사용합니다.

### 2. 명확한 액션 제공

카드가 클릭 가능하면 전체가 클릭 영역이 되도록 하고, 호버 시 시각적 피드백을 제공합니다.

### 3. 적절한 간격 유지

카드 간 충분한 여백을 두어 시각적으로 구분되도록 합니다.

### 4. 반응형 디자인

화면 크기에 따라 카드 개수와 크기를 조정합니다.
