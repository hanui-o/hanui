# 이어서 작업하기 (핸드오프)

> VSCode의 Claude Code(또는 새 세션)에서 이 책 작업을 이어갈 때 이 파일부터 읽으면 된다.
> 작성 환경 조건: **Claude Code**여야 함 (CLAUDE.md·`/blog` 커맨드·프로젝트 메모리를 읽는 도구). Cursor·일반 챗봇은 맥락이 안 따라옴.

## 프로젝트 한 줄

책 **「퇴근 30분 빨라지는 AI」**(가제)를 블로그 글로 매일 한 꼭지씩 쓰며 초고를 만든다.
하루 한 꼭지 = 책 한 꼭지.

## 먼저 읽을 문서 (이 순서)

1. `docs/book/기획서.md` — 타겟·컨셉·차별점·목차·리스크 (마스터)
2. `docs/blog/content-plan.md` — 목차 = 주제 큐 + 이번 주 일정 + 글 상태
3. `.claude/commands/blog.md` — `/blog` 커맨드(글 작성 규칙·독자 모드)
4. 프로젝트 메모리는 Claude Code가 자동 로드 (`book-ai-not-scary`)

## 핵심 결정 (요약)

- **타겟:** 글은 경리에게, 책은 사무직 전체에 판다.
- **컨셉:** 코딩 없이 AI한테 말 걸어 **업무 도구를 직접 만든다**(물어보기→만들기). 파는 건 버튼이 아니라 **판단**.
- **도구:** 클로드 Artifacts(브라우저). "코딩" 단어 안 씀.
- **톤:** 쉬운 말·비유. 글 끝마다 "오늘 딱 하나" + "이 장의 판단 1가지".
- **금지:** 주식·투자 등 금융 조언성 콘텐츠.

## 자동 발행 시스템 (중요)

**글은 시간 날 때 몰아서 쓰고, 발행은 매일 자동.** 직접 발행 버튼 누를 필요 없다.

- **예약 큐:** `apps/docs/blog-queue/` 에 mdx를 두면 사이트에 안 보인다(대기).
- **자동 발행:** 매일 09:10 KST, GitHub Actions(`.github/workflows/daily-publish.yml`)가 큐에서
  `date`가 오늘 이하인 글을 `apps/docs/src/content/blog/`로 옮겨 커밋·푸시 → Vercel 자동 배포.
- **재고 알림:** 남은 예약글 < 3편이면 GitHub 이슈(`blog-restock` 라벨) 자동 생성.
- **쟁여두는 법:** 큐에 글 작성 + `date`를 미래로(하루 간격) + 커밋·푸시. 끝. 자세히는 `apps/docs/blog-queue/README.md`.
- **로컬 점검:** `DRY_RUN=1 node scripts/publish-due-posts.mjs`
