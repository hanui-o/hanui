# 출간기획서 (초안)

> 블로그 연재 = 집필. 매일 한 꼭지가 책 한 꼭지가 된다.
> 진행 상황·주제 큐는 `docs/blog/content-plan.md` 참고.

## 한 줄 컨셉

**"코딩 한 줄 없이, AI한테 말만 걸어 사무직이 자기 업무 도구를 직접 만든다."**

설명서(물어보기)가 아니라 **만들기**. 활용서가 아니라 **자동화**.
그리고 파는 건 버튼이 아니라 **판단** — "AI한테 뭘 맡기고, 뭘 내 눈으로 지킬지." 그래서 AI가 좋아져도 안 낡는다.

## 가제

**「퇴근 30분 빨라지는 AI」** — 부제: 코딩 없이, 말로 만드는 나의 업무 도구

대안: 「내 업무를 자동화했습니다」 / 「말로 만드는 업무 도구」 / 「AI한테 시켰더니 퇴근이 빨라졌다」

## 타겟 독자

**쓰는 대상(글) = 경리 / 파는 대상(시장) = 사무직 전체.**
한 사람(예: 소규모 회사 경리 "김 대리")의 구체적 고통에 대고 쓴다 → 글이 생생해지고, 옆자리 총무·서무·관리직도 "내 얘기네" 하고 산다. 좁게 쓰되 넓게 팔린다.

- **핵심 페르소나:** 소규모 회사 경리·사무 담당. 엑셀 장부·정산·거래처 메일·영수증 정리·월 마감에 매일 시달림.
- AI는 들어봤고 한두 번 써봤지만 "물어보기"에서 멈춘 사람. "내 일에 어떻게?"가 안 풀린 사람.
- 확장 독자: 총무·서무·관리·일반 사무직. + 회사 교육/팀 단위 구매 가능성.

### 시장 근거 (검증됨)

- AI 도서 독자의 **78.5%가 30~50대** (40대 30.5%, 50대 24.2%).
- 50대 챗GPT **인지율 33% vs 이용률 17.7%** → "아는데 못 써먹는" 격차.
- 바이브코딩 = 2026년 **$4.7B, 사용자 63%가 비개발자**.
- **젠슨 황 유퀴즈(2026-06-10, 5.7%)**: "AI는 쉽다, 원하는 걸 말하면 된다" → 전 국민 데워진 직후.
- "직장인 업무"는 유행이 아니라 **상시 수요** — 증시·부업처럼 식거나 사기 이미지 없음(저자·출판사 리스크 최저).

## 기획 의도 — 빈자리

| 기존 부류                 | 한계                                             |
| ------------------------- | ------------------------------------------------ |
| 교양·전망서 (AI 2026 등)  | "왜 중요한지"만, 손에 안 잡힘                    |
| AI 활용서 (프롬프트 모음) | "물어보기"에서 끝. 내 업무에 안 박힘             |
| 혼공 바이브코딩           | "웹앱 만들기" = 개발 입문 톤, 직장인 업무가 아님 |
| AI 주식·부업서            | 권위 필요 + 금융/과장 리스크 + 유행              |

→ 빈자리: **직장인이 "물어보기"를 넘어 "내 업무 도구를 만들어" 시간을 버는** 책.

## 차별점 (한 문장)

- 활용서와 다름: **물어보기 → 만들기**, 프롬프트 모음 → **사고법(판단)**
- 혼공과 다름: **웹앱(개발) → 업무 도구(목적)**, 터미널 → **클로드 Artifacts(브라우저)**
- "코딩"이라는 단어를 안 쓴다. **"말로 만들기"**
- 경리 특화: **숫자는 AI를 안 믿고 검산** — 정확성이 생명인 직무에 맞춘 신뢰 기준

## 저자 강점 ("왜 당신?")

매일 AI(클로드)로 실제 도구를 만드는 개발자/빌더이면서, 그걸 **비개발자 언어로 번역**할 수 있다. 투자서처럼 없는 권위를 빌릴 필요가 없다 — 만드는 게 일상이라 곧 자기 경험.

## 도구 선택: 클로드 Artifacts

- 브라우저에서 채팅만으로 **작동하는 도구가 즉시 보임** (터미널·설치·배포 없음)
- 추가 비용·셋업 없음 → iteration tax 최소 → "쉽다" 약속 유지
- 혼공(클로드 코드)과 자연 차별화
- 더 키우려는 독자에겐 후반부에 Lovable/Bolt로 다리

## 목차 (업무별 — 각 장에 "결과물" + "이 장이 가르치는 판단 1가지")

구조 원칙: **앱이 아니라 경리의 반복 업무(고통)로 묶는다.** 각 장은 작동하는 도구 하나 + 그 도구를 안전하게 쓰는 판단 한 조각.

**1부. 시작 — 무서움 빼고, 사고법 심기**

| #   | 제목                                   | 결과물    | 이 장의 판단                   |
| --- | -------------------------------------- | --------- | ------------------------------ |
| 1   | AI는 말귀 알아듣는 인턴이다            | (안심)    | AI를 동료로 보는 눈            |
| 2   | 부탁하듯 말 거는 법                    | 첫 부탁   | 좋은 부탁 = 구체적인 부탁      |
| 3   | "물어보기" vs "만들기"                 | (개념)    | 한 번 만들어 백 번 쓰기        |
| 4   | AI를 믿는 선 — 맡길 것 vs 내가 지킬 것 | (핵심 장) | **숫자·사실은 AI를 안 믿는다** |

**2부. 경리·사무 업무를 도구로** (각 1꼭지, 작동하는 결과물)

| #   | 업무 고통                   | 결과물(도구)                  | 관련 앱   | 이 장의 판단             |
| --- | --------------------------- | ----------------------------- | --------- | ------------------------ |
| 5   | 매번 비슷한 메일·안내문     | 칸 채우면 완성되는 작성 도구  | 메일·워드 | 상대·상황 따라 톤 고르기 |
| 6   | 표 정리·합계·집계           | 붙여넣으면 정리·계산하는 도구 | 엑셀      | **합계는 눈으로 검산**   |
| 7   | 영수증·내역을 표로          | 붙여넣어 항목 분류·정리       | 엑셀      | 빠진 항목 잡아내기       |
| 8   | 길어서 못 읽음(메일·회의록) | 요약 + 할 일 추출             | 메일·노션 | 요약이 버린 것 확인      |
| 9   | 주간·월간 보고              | 보고 초안 생성기              | 워드      | 사실과 추측 구분         |
| 10  | 공지·양식 만들기            | 안내문·양식 한 장             | 워드·한글 | 회사 형식에 맞추기       |

**3부. 막혀도 OK / 다음 걸음**

| #   | 제목                              | 핵심                |
| --- | --------------------------------- | ------------------- |
| 11  | 이상하게 나올 때 고쳐달라 하는 법 | iteration tax 대응  |
| 12  | 동료·팀에 공유하고 자랑하기       | 혼자 → 팀으로       |
| 13  | 더 큰 걸 만들고 싶어지면          | Lovable/Bolt 맛보기 |

> 파포(PPT)·지라는 1차 제외. 파포는 "더 키우기"나 부록, 지라는 사무직 일부만 해당해 제외.

## 집필 = 연재 매핑

- 블로그 시리즈명(frontmatter): **`퇴근 30분 빨라지는 AI`** (제목 확정 시 일괄 변경)
- 하루 한 꼭지 발행 → 13편이면 1차 원고 골격 완성
- 일반인 모드(쉬운 말), 글 끝마다 "오늘 딱 하나" 실천
- 각 글에 "이 장의 판단 1가지"를 반드시 넣어 활용서로 안 굴러떨어지게 한다
