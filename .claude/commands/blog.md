hanui 블로그 글을 컨벤션대로 작성해줘.

## Arguments

- `$ARGUMENTS`: 글 주제 (예: `Claude Code로 테스트 자동화`, `KRDS 색상 토큰 설계`)
  - 비어 있으면 사용자에게 "오늘 어떤 주제로 쓸까요?" 묻고, 최근 글 목록(`apps/docs/src/content/blog/`)을 보여주며 중복되지 않는 방향을 1~2개 제안한다.

## 사전 확인 (작성 시작 전)

1. **글 종류 판별** — 두 가지뿐:
   - **신규 글** (기본): `apps/docs/src/content/blog/[slug].mdx` 하나만 생성
   - **velog 이전 글**: 사용자가 "velog에서 옮긴다/이전"이라고 명시할 때만. 이 경우 [CLAUDE.md](CLAUDE.md) 규칙대로 **반드시 두 파일**:
     1. `apps/docs/src/content/blog/[slug].mdx`
     2. `docs/blog/velog/[다음번호]-[slug]-notice.md` (번호는 `docs/blog/velog/`의 최대 번호 + 1)
   - 어느 쪽인지 불확실하면 사용자에게 확인한다.

2. **중복 체크**: `apps/docs/src/content/blog/`에 비슷한 주제·슬러그가 이미 있는지 확인. 있으면 알리고 진행 여부를 묻는다.

3. **슬러그 결정**: 영문 kebab-case, 주제를 압축 (예: `claude-code-test-automation`). 기존 슬러그 네이밍 관습을 따른다 (`tips-*`, `a11y-*`, `claude-code-*` 등 접두사 활용).

## mdx 작성 규칙

### Frontmatter (필수)

```yaml
---
title: '한국어 제목 — 핵심을 찌르는 부제'
description: '검색·미리보기용 1~2문장 요약. 글의 결론이나 후크를 담는다.'
date: 'YYYY-MM-DD' # 오늘 날짜
tags: ['태그1', '태그2', '태그3'] # 한국어/영어 혼용 OK, 3~5개
author: 'odada'
---
```

- 선택 필드: `series`, `seriesOrder` (시리즈 글일 때), `image`, `canonicalUrl`. blog.ts가 읽는 필드만 사용.
- `date`는 오늘 날짜를 실제로 확인해서 넣는다.

### 독자 모드 먼저 정한다

글마다 독자가 다르므로 둘 중 하나를 먼저 고른다 (주제·시리즈로 자동 판단, 애매하면 사용자에게 확인):

- **일반인 모드** _(기본 — 책 「AI, 안 무서워요」 등 비개발자 대상)_: 코드·전문용어 없음. 용어가 꼭 필요하면 즉시 일상어로 풀이. 비유와 이야기 중심. 말 걸듯 친근하게. 안심시키는 톤. 글 끝에 **"오늘 딱 하나"** 식 실천 한 줄. `### 자료`/`## 반대 의견` 같은 딱딱한 섹션은 생략 가능.
- **전문가 모드** _(개발자 대상 — Claude Code·HANUI·접근성 등)_: 아래 톤 규칙을 그대로 적용.

### 본문 톤 (전문가 모드 기준 — 일반인 모드는 위 규칙 우선)

- **구체적인 장면으로 연다.** 추상적 정의 X. ("테스트 38개를 한 번에 짜야 했다." / "월요일에 통과한 버튼이 금요일에 항의 메일로 돌아왔어요.")
- 짧고 단정한 문장. 군더더기 없이. 숫자·사건·반박으로 밀고 나간다.
- `## 소제목`으로 단락 구분. 코드/표/링크 적극 사용.
- 후반부에 자주 쓰는 구조: `## 반대 의견`(예상 반론 + 답), `## 닫기`(질문 몇 개로 마무리).
- 맨 끝 `### 자료` 섹션에 관련 링크 (hanui repo, 외부 출처 등).

### 공통 (두 모드 모두)

- **구체적인 장면·말로 연다.** 추상적 정의로 시작하지 않는다.
- AI 글 티 나는 표현 금지: "~을 살펴보겠습니다", "결론적으로", 과한 이모지·불릿 남발 X. 사람이 쓴 호흡으로.
- 책 시리즈 글이면 끝에 다음 글을 예고해 연결한다.

### velog 이전 공지 형식 (이전 글일 때만)

```markdown
# [원본 제목]

> 이 글의 원본은 **<a href="https://hanui.io/blog/[slug]" target="_blank" rel="noopener noreferrer">hanui 블로그</a>로 이전**했어요.
> 더 나은 가독성과 최신 내용은 원본 링크에서 확인해주세요!
>
> 👉 https://hanui.io/blog/[slug]

---

[원본 글 첫 두세 문단 요약]

전체 내용은 hanui 블로그에서 확인하세요 → **https://hanui.io/blog/[slug]**
```

## 마무리

1. 작성한 파일 경로를 명확히 보고한다.
2. 로컬 확인 방법 안내: `apps/docs`에서 `pnpm dev` → `/blog/[slug]` 접속.
3. **커밋·푸시는 사용자 확인 후에만.** 자동으로 커밋하지 않는다. (커밋 메시지는 Conventional Commits, 한국어 OK: `feat: [제목] 블로그 글 추가`)
