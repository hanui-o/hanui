UI 코드를 웹 디자인 가이드라인 기준으로 리뷰해줘.

## Arguments

- `$ARGUMENTS`: 리뷰할 파일 또는 패턴 (없으면 변경된 파일 대상)

## 점검 항목

1. **레이아웃 & 반응형:**
   - 모바일 퍼스트 접근
   - 고정 px 대신 상대 단위 (rem, em, %) 사용
   - 미디어 쿼리 일관성
   - 컨테이너 max-width 설정

2. **인터랙션:**
   - hover/focus/active 상태 정의
   - 터치 타겟 최소 44x44px
   - 클릭 가능 요소에 cursor: pointer
   - 트랜지션/애니메이션 자연스러움 (200-300ms)

3. **타이포그래피:**
   - 폰트 크기 체계 일관성
   - line-height 가독성 (1.4-1.6)
   - 최대 줄 길이 (60-80ch)

4. **색상 & 테마:**
   - 디자인 토큰/CSS 변수 사용 여부
   - 하드코딩된 색상값 지양
   - 다크모드 대응 (해당 시)
   - 충분한 색상 대비

5. **컴포넌트 패턴:**
   - 로딩 상태 처리
   - 에러 상태 처리
   - 빈 상태 (empty state) 처리
   - 스켈레톤 UI 적용 여부

## Output

- 항목별 pass/fail 표시
- 스크린샷 또는 코드 기반으로 구체적 개선안 제시
