# 에러 수정 명령어

**중요 규칙:**

1. **절대 기존 코드를 삭제하지 마세요**
   - 에러 수정이 어렵다고 해서 작성된 코드를 삭제하는 것은 절대 금지입니다
   - 모든 기존 작업 내용을 유지해야 합니다

2. **절대 다른 내용으로 변경하지 마세요**
   - 에러를 수정하기 위해 완전히 다른 구현으로 바꾸는 것은 금지입니다
   - 현재 구현 방식을 최대한 유지하면서 에러만 수정해야 합니다

3. **절대 과거로 되돌리지 마세요**
   - 이전 버전으로 롤백하거나 되돌리는 것은 금지입니다
   - 현재 상태에서 에러만 수정해야 합니다

**에러 수정 원칙:**

- 현재 작업 상태를 100% 유지
- 최소한의 변경으로 에러만 해결
- 기존 로직과 구조 완전히 보존
- 파일 경로, import 경로, 타입 에러 등만 수정

**작업 방법:**

1. 에러 메시지를 정확히 분석
2. 에러가 발생한 정확한 위치 파악
3. 기존 코드를 최대한 유지하면서 최소한의 수정만 진행
4. 수정 후 다른 부분에 영향이 없는지 확인

현재 발생한 에러를 위 원칙에 따라 수정해주세요.
