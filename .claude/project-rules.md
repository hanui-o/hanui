# HANUI 프로젝트 규칙

## ⛔ 절대 금지 사항

### 1. 파일/디렉토리 임의 삭제 금지

- **절대 임의로 파일이나 디렉토리를 삭제하지 마세요**
- 특히 다음 항목들은 절대 삭제 금지:
  - 컴포넌트 파일 (`.tsx`, `.ts`)
  - 문서 페이지 (`apps/docs/src/app/**/*.tsx`)
  - 설정 파일 (`.json`, `.config.*`)
  - 스타일 파일 (`.css`, `.scss`)

### 2. 삭제가 필요한 경우

- **반드시 사용자에게 먼저 물어보세요**
- 삭제 이유를 명확히 설명하세요
- 사용자의 명시적인 승인을 받은 후에만 삭제하세요

### 3. GitHub Actions 오류 처리

- GitHub Actions에서 lint/build 오류가 발생했을 때:
  - ❌ 오류 원인 파일을 바로 삭제하지 마세요
  - ✅ 먼저 오류를 수정하려고 시도하세요
  - ✅ 수정이 어려우면 사용자에게 상황을 설명하고 의견을 물어보세요

### 4. 실험적 코드

- "실험적"이라는 이름이나 주석이 있어도:
  - 사용자가 만든 코드일 수 있습니다
  - 임의로 삭제하지 마세요
  - 삭제 전에 반드시 확인하세요

## ✅ 권장 사항

### 1. 변경 전 확인

```
1. 파일이 언제 만들어졌는지 확인 (git log)
2. 파일이 어디서 사용되는지 확인 (grep, imports)
3. 사용자에게 변경/삭제 의도 확인
```

### 2. 오류 해결 우선순위

```
1순위: 코드 수정으로 해결
2순위: 사용자에게 문제 설명 및 해결 방안 제시
3순위: 사용자 승인 후 삭제/변경
```

### 3. 커밋 메시지

- 삭제된 파일이 있다면 왜 삭제했는지 명확히 기록
- "fix", "refactor" 등의 타입 정확히 사용

## 📝 예외 사항

다음 항목들은 임의로 삭제 가능:

- `.DS_Store` 파일
- `node_modules/` (단, package.json은 삭제 금지)
- `dist/`, `build/` 등 빌드 결과물
- 명백한 임시 파일 (`*.tmp`, `*.temp`)

## 🔍 확인 절차

파일 삭제/변경 전 체크리스트:

- [ ] 이 파일이 무엇을 하는지 이해했는가?
- [ ] 이 파일이 다른 곳에서 사용되는지 확인했는가?
- [ ] 이 파일을 누가, 언제 만들었는지 확인했는가? (git log)
- [ ] 사용자에게 변경/삭제 의도를 물어봤는가?
- [ ] 대안이 있는가?
